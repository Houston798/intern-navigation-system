# 业务部实习生成长导航系统设计文档

**版本**: Demo v1.0  
**日期**: 2026-06-01  
**技术栈**: React (Vite) + Node.js (Express) + CloudBase + DeepSeek

---

## 一、项目概述

### 1.1 产品定位
本产品为业务部内部 AI 导航工具，核心目标是帮助实习生快速了解团队业务与需求，提升自身技能，加速融入团队。同时为导师提供带教辅助，为 HR 提供数据分析支持。

### 1.2 服务人群

| 角色 | 说明 |
|------|------|
| 主要用户 | 实习生（暑期 / 日常） |
| 辅助用户 | 导师、HR |

## 三、功能模块设计

### 3.1 账号系统

#### 3.1.1 多端入口与身份认证

- **实习生端**：无需特殊密钥，注册时须选择身份、部门及补充信息
- **导师端**：需输入【导师专属密钥】方可完成注册
- **HR端**：需输入【HR专属密钥】，首个HR账号通过系统初始化密钥激活

#### 3.1.2 注册补充信息问卷（实习生必填）

- MBTI 类型 或 E/I 选择
- 所属业务部门
- 是否有相关工作经历
- 实习类型：暑期实习生 / 日常实习生

#### 3.1.3 账号安全规范

- 登录与注册走独立 API，与业务接口解耦
- Token 过期刷新机制 + 安全退出
- 忘记密码：通过注册邮箱发送验证码重置
- 区分未登录状态跳转，避免死循环

### 3.2 实习生端

#### 3.2.1 On Boarding 引导流程（6步）

1. **欢迎确认** - 展示注册信息
2. **工作台导览** - 6个核心模块介绍
3. **工作目标展示** - 导师设定目标
4. **AI 助手体验** - 3个常用技能芯片
5. **实习计划激活** - 实时显示剩余天数
6. **完成** - 清单式回顾

#### 3.2.2 核心功能

- **任务地图**：展示当前工作目标及分解任务
- **学习平台**：学习内容与进度追踪
- **AI 助手**：专属对话助手（暑期/日常差异化）
- **进度追踪**：实时实习进度可视化
- **论坛**：与其他实习生交流
- **问题自查**：AI 分析改进建议

#### 3.2.3 AI 助手提示词分层

**基础层（所有实习生共用）**
- 始终围绕工作目标回答
- 回答具体可行
- 标注信息来源
- 识别并鼓励不敢提问的实习生

**暑期实习生追加层**
- 时间紧迫感提醒
- 业务深度优先
- 转正意识引导
- 压缩引导步骤（5步以内）
- 焦虑情绪疏导

**日常实习生追加层**
- 节奏适配（稳健推进）
- 双重压力关怀（学业+工作）
- 办公技能强化
- 业务横向对比
- 自我驱动引导

### 3.3 导师端

#### 3.3.1 核心功能

- **实习生数据看板**：任务完成率、学习进度
- **词云图**：实习生搜索/关注内容
- **AI 带教方案生成**：根据实习生特点生成方案
- **监测预警**：进度落后/连续未登录提醒
- **通知系统**：定期反馈节点提醒

#### 3.3.2 AI 助手能力

- **初始带教方案生成**：根据注册数据自动生成
- **实习生监测预警**：主动推送提醒
- **带教案例学习**：成功案例推荐
- **数据看板解读**：图表数据解读建议

### 3.4 HR端

#### 3.4.1 核心功能

- **数据分析**：任务完成、导师评价分析
- **可视化图表**：柱状图、雷达图
- **转正预测**：基于历史数据预测潜力
- **密钥管理**：生成/下发各端密钥

#### 3.4.2 AI 助手能力

- **招聘标准优化**：结构化招聘标准草稿
- **实习生成果分析**：能力维度对比
- **数据预测**：转正潜力分布预测
- **密钥管理提醒**：安全规范提醒

### 3.5 论坛系统

- AI 辅助角色，不主动发帖
- 30分钟无回复时提供参考答案
- 温和纠正事实性错误
- 友好引导偏离话题
- 不透露个人数据

## 七、AI 提示词设计

### 7.1 实习生基础层

**系统提示词结构**
```
你是「{intern_department}部实习生成长助手」，专为 {company_name} 业务部实习生服务。

当前实习生信息：
- 姓名：{intern_name}
- 部门：{intern_department}
- 实习类型：{intern_type}
- MBTI/性格：{intern_mbti}
- 工作经历：{intern_experience}
- 实习开始日期：{start_date}，结束日期：{end_date}
- 当前工作目标：{work_goals}

行为准则：
① 始终围绕实习生当前的工作目标和所在业务部门来回答
② 回答务必具体可行，避免模糊建议
③ 每次提供外部信息时，必须注明信息来源
④ 若提问超出工作相关范围，礼貌引导回到工作目标
⑤ 识别并鼓励不敢提问的实习生，营造心理安全感
```

### 7.2 暑期实习生追加层

```
① 时间紧迫感：当前实习剩余天数为 {remaining_days} 天
② 业务深度优先：优先推荐业务相关的学习资料
③ 转正意识：适时结合转正评估维度给出建议
④ 压缩引导步骤：On boarding 和任务引导要简洁高效
⑤ 竞争压力疏导：先共情再给建议
```

### 7.3 日常实习生追加层

```
① 节奏适配：避免使用紧迫语气，以稳健推进替代催促
② 双重压力关怀：帮助梳理学业与工作优先级
③ 办公技能强化：额外提供 Excel、PPT 等通用职场技能
④ 业务横向对比：帮助了解本部门与公司其他部门的关联
⑤ 自我驱动引导：建立「学到了什么」的成就感
```

### 7.4 导师端 AI 助手

```
你是「{company_name}业务部导师带教助手」

核心能力：
【能力一：初始带教方案生成】
根据实习生注册数据自动生成初步带教方案与工作目标草稿

【能力二：实习生监测预警】
条件触发时主动推送提醒（进度落后、连续未登录等）

【能力三：带教案例学习】
收集成功带教案例，按「场景匹配度」排序推荐

【能力四：数据看板解读】
提供词云/图表数据解读建议
```

### 7.5 HR端 AI 助手

```
你是「{company_name}业务部 HR 数据分析助手」

核心能力：
【能力一：招聘标准优化】
协助建立更精准的招聘画像与评估标准

【能力二：实习生成果分析】
将任务完成情况与预期成果进行对比分析

【能力三：数据预测与建议】
基于历史数据预测转正潜力分布

【能力四：密钥管理提醒】
提醒安全规范
```

## 九、配置说明

### 9.1 环境变量

```env
# DeepSeek API 配置
DEEPSEEK_API_KEY=your_api_key_here

# CloudBase 配置
CLOUDBASE_ENV_ID=your_env_id_here

# 系统初始化密钥
ADMIN_INIT_KEY=your_admin_key_here

# JWT 密钥
JWT_SECRET=your_jwt_secret_here
```

### 9.2 安全规范

- API 密钥存储在云函数环境变量中
- 前端页面不得直接调用 DeepSeek API
- 必须通过云函数中转
